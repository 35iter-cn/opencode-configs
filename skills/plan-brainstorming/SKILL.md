# Brainstorming Principles for Planning

## Core Constraints

1. **No implementation without approval** — If the design/spec is not yet user-approved, STOP planning and engage the user in brainstorming before proceeding
2. **Explore alternatives** — For any significant architectural or approach decision, present 2-3 options with trade-offs and your recommendation
3. **One question at a time** — When seeking user input on plan decisions, ask one focused question per message
4. **YAGNI ruthlessly** — Remove unrequested features from all plans. Scope creep is the enemy.
5. **Incremental validation** — Present plan sections to the user for approval before proceeding to the next section
